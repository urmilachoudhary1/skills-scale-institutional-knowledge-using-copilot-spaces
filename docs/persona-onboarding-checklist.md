# Persona Onboarding Checklist

Use this checklist to ensure every new core team member is set up for success and clarity in their role on OctoAcme projects.

## All Personas
- [ ] Review project charter and key program process docs
- [ ] Introductory meeting with Project Manager for role expectations
- [ ] Access to project boards and communication tools set up
- [ ] Added to relevant recurring meetings and comms channels

## Role-Specific Onboarding

### For Developers
- [ ] Review coding conventions and branch/PR workflow documentation
- [ ] Set up local environment and access to CI systems

### For Product Managers
- [ ] Review product vision, backlog structure, and prioritization practice
- [ ] Walkthrough of metrics and success measurement tools

### For Project Managers
- [ ] Familiarize with project planning templates and reporting cadence
- [ ] Walkthrough of risk and stakeholder management practices

### For Business Analyst
- [ ] Review requirements documentation templates and acceptance criteria processes
- [ ] Schedule requirements workshop with Product Manager

### For UX Designer
- [ ] Access to design systems, assets, and research findings
- [ ] Schedule usability review with Product Manager

### For DevOps Engineer
- [ ] Access to CI/CD pipelines, infrastructure code, and monitoring dashboards
- [ ] Review deployment, rollback, and incident playbooks

### For Customer Support Lead
- [ ] Review feedback loops, issue tracking, and release communication processes
- [ ] Introduced to support documentation and customer contact channels

## Completion
- [ ] First sprint retrospective attended by new team member
- [ ] Feedback on onboarding process shared with Project Manager
